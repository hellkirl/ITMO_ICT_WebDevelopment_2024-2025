    Веб-приложение "Автогонки"

    Структура Проекта

    Проект состоит из следующих основных компонентов:

    • Модели: Определяют структуру данных и связи между различными сущностями.
    • Формы: Обеспечивают интерфейс для ввода данных пользователями.
    • Представления (Views): Обрабатывают запросы пользователей и возвращают соответствующие ответы.
    • Шаблоны (Templates): Определяют внешний вид страниц приложения.
    • URL-конфигурация: Определяет маршруты для различных представлений.
    • Миграции: Управляют изменениями в базе данных.

    Модели

    Модель Race

    Представляет информацию о гонке.

    • Поля:
      • name: Название гонки.
      • date: Дата проведения гонки.
      • time: Время проведения гонки.
      • result: Результат гонки.

    Модель Registration

    Отвечает за регистрацию пользователей на гонки.

    • Поля:

      • race: Связь с моделью Race.
      • user: Связь с моделью User (стандартная модель пользователей Django).

    • Особенности:

      • Уникальное ограничение на комбинацию race и user, что предотвращает множественную регистрацию одного пользователя на одну гонку.

    Модель Comment

    Позволяет пользователям добавлять комментарии к гонкам.

    • Поля:
      • race: Связь с моделью Race.
      • user: Связь с моделью User.
      • date: Дата добавления комментария (автоматически устанавливается при создании).
      • text: Текст комментария.
      • comment_type: Тип комментария (положительный или отрицательный).
      • rating: Оценка гонки.

    Формы

    Форма RegistrationForm

    Используется для регистрации пользователя на гонку.

    • Поля:

      • race: Выбор гонки для регистрации.

    • Особенности:

      • Проверяет, зарегистрирован ли пользователь уже на выбранную гонку, и предотвращает повторную регистрацию.

    Форма CommentForm

    Используется для добавления комментариев к гонкам.

    • Поля:
      • race: Выбор гонки, к которой добавляется комментарий.
      • text: Текст комментария.
      • comment_type: Тип комментария.
      • rating: Оценка гонки.

    Представления (Views)

    Представление race_list

    Отображает список всех гонок и позволяет пользователю регистрироваться на них.

    Представление register_racer

    Обрабатывает регистрацию пользователя на выбранную гонку.

    • Логика:
      • При получении POST запроса извлекает race_id из данных формы.
      • Проверяет, зарегистрирован ли пользователь уже на выбранную гонку.
      • Если нет, создает новую запись регистрации и уведомляет пользователя об успешной регистрации.
      • Если пользователь уже зарегистрирован, отображает сообщение об ошибке.

    Представление race_detail

    Отображает подробную информацию о конкретной гонке, включая список участников и комментарии.

    Представления edit_registration и delete_registration

    Позволяют пользователям редактировать или удалять свои регистрации на гонки.

    Представление add_comment

    Позволяет пользователям добавлять комментарии к гонкам.

    Шаблоны (Templates)

    Шаблон base.html

    Базовый шаблон, содержащий общие элементы страницы, такие как заголовок, навигация и футер. Также включает блок для отображения сообщений пользователям.

    Шаблон race_list.html

    Отображает список всех гонок с возможностью регистрации на каждую гонку через кнопку "Зарегистрироваться". Если пользователь уже зарегистрирован на гонку, отображаются кнопки для редактирования и удаления регистрации.

    Шаблон race_detail.html

    Отображает подробную информацию о гонке, включая список участников и комментарии. Позволяет авторизованным пользователям добавлять комментарии.

    Шаблон register_racer.html

    Отображает форму регистрации на гонку. Однако в текущей реализации регистрация осуществляется непосредственно через кнопки в списке гонок, поэтому этот шаблон может быть опциональным.

    URL-конфигурация

    Файл races/urls.py

    Определяет маршруты для различных представлений приложения.

    • /: Список гонок (race_list).
    • /register/: Регистрация на гонку (register_racer).
    • /race/<int:pk>/: Детали гонки (race_detail).
    • /race/<int:pk>/edit/: Редактирование регистрации (edit_registration).
    • /race/<int:pk>/delete/: Удаление регистрации (delete_registration).
    • /comment/add/: Добавление комментария (add_comment).

    Файл racing_project/urls.py

    Включает маршруты приложения races и маршруты для аутентификации пользователей.

    Миграции

    После создания и изменения моделей были созданы миграции для обновления структуры базы данных. Миграции включают:

    • Создание таблиц для моделей Race, Registration и Comment.
    • Добавление уникального ограничения на модель Registration.
    • Обновление полей модели Race для установки значений по умолчанию и предотвращения NULL значений.

    Функциональное тестирование

    После реализации всех функций было проведено тестирование для проверки корректности работы приложения:

    1. Регистрация на гонку:

      • Авторизованный пользователь успешно регистрируется на гонку через кнопку "Зарегистрироваться".
      • При попытке повторной регистрации отображается сообщение об ошибке.

    2. Отображение участников:

      • В детальной информации о гонке отображаются имена всех зарегистрированных участников.

    3. Добавление комментариев:

      • Авторизованный пользователь может добавлять комментарии к гонкам.
      • Комментарии отображаются с именами пользователей и датой добавления.

    4. Редактирование и удаление регистраций:

      • Пользователь может редактировать или удалять свои регистрации на гонки через соответствующие кнопки.

    Заключение

    В результате выполнения лабораторной работы было создано веб-приложение "Автогонки" с использованием Django. Приложение позволяет пользователям просматривать гонки, регистрироваться на них, добавлять комментарии и управлять своими регистрациями.
