    Безопасность

    В проекте используется JWT для авторизации на эндпоинтах.

    Описание эндпоинтов

    Диагнозы

    GET /api/v1/diagnoses/

    • Аргументы: Отсутствуют
    • Описание: Получает список всех диагнозов.

    POST /api/v1/diagnoses/

    • Аргументы:
    • Тело запроса:
      • Name (строка, обязательное): Название диагноза.
      • IllnessType (строка, необязательное): Тип заболевания.
      • Description (строка, необязательное): Подробное описание.
      • GeneralTreatmentRecommendations (строка, необязательное): Общие рекомендации по лечению.
    • Описание: Создает новую запись диагноза.

    GET /api/v1/diagnoses/{DiagnosisID}/

    • Аргументы:
    • Путь:
      • DiagnosisID (целое число, обязательное): Уникальный идентификатор диагноза.
    • Описание: Получает детали конкретного диагноза.

    PUT /api/v1/diagnoses/{DiagnosisID}/

    • Аргументы:
    • Путь:
      • DiagnosisID (целое число, обязательное): Уникальный идентификатор диагноза.
    • Тело запроса:
      • Name (строка, обязательное)
      • IllnessType (строка, необязательное)
      • Description (строка, необязательное)
      • GeneralTreatmentRecommendations (строка, необязательное)
    • Описание: Полностью обновляет существующий диагноз.

    PATCH /api/v1/diagnoses/{DiagnosisID}/

    • Аргументы:
    • Путь:
      • DiagnosisID (целое число, обязательное)
    • Тело запроса: Частичные поля для обновления.
    • Описание: Частично обновляет существующий диагноз.

    DELETE /api/v1/diagnoses/{DiagnosisID}/

    • Аргументы:
    • Путь:
      • DiagnosisID (целое число, обязательное)
    • Описание: Удаляет конкретный диагноз.
